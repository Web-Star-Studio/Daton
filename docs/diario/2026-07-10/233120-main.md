## Obrigatoriedade de treinamento — seleção de múltiplos cargos

**O que foi feito:** No diálogo de obrigatoriedade de treinamento, a criação passou a
aceitar **vários cargos de uma vez**. Cada cargo selecionado gera uma obrigatoriedade
(mantendo o modelo de "uma regra por cargo"), tudo num único "Salvar". Foi incluído o
recurso **"Selecionar todos os encontrados"**, que respeita a busca: ao digitar
"ANALISTA", por exemplo, um clique marca todos os cargos de analista. A edição continua
sendo de um cargo por regra.

**Por quê:** Pedido da cliente (Ana Corrêa / Gabardo) para aplicar um mesmo treinamento
a grupos inteiros de cargos (ex.: todos os analistas, todos os ajudantes) sem cadastrar
regra por regra.

**Impacto / área:** Módulo de Aprendizagem (Obrigatoriedades). Reutiliza o serviço já
existente, que evita duplicidade (não recria uma regra que já exista). Em caso de falha
parcial ao criar em lote, o diálogo permanece aberto para o usuário reter apenas os que
faltaram, sem perder a seleção.

**Status:** Concluído e publicado (PR #148, integrado à branch principal).

**Validações:** Verificação automática de tipos do projeto sem erros; lógica de criação
em lote coberta por testes automatizados (14 casos); comportamento validado em navegador
contra base de dados de teste (criação em lote, "selecionar todos", tratamento de
duplicados e edição).
